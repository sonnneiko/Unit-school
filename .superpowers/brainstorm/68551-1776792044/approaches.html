Как реализовать отдельный сайдбар для админки?

Три архитектурных подхода. Нажми чтобы выбрать.

A

Отдельный AdminAppLayout (рекомендую)

Admin-маршруты выносятся из текущего AppLayout в собственный AdminAppLayout с AdminSidebar. Два полностью независимых дерева: / → AppLayout+Sidebar, /admin → AdminAppLayout+AdminSidebar.

Плюсы

  • Чистое разделение
  • Обе навигации независимы
  • Легко развивать каждую отдельно

Минусы

  • Нужно создать новый компонент
B

Умный Sidebar по маршруту

Существующий Sidebar определяет текущий маршрут: если /admin/* — показывает админ-пункты, иначе — обычные. Один компонент, разная конфигурация.

Плюсы

  • Меньше новых файлов

Минусы

  • Один компонент знает о двух режимах
  • Сложнее поддерживать
C

AdminSidebar внутри AdminLayout

Оставляем admin-маршруты внутри AppLayout, но AdminLayout (уже существует) сам рендерит свой сайдбар поверх/вместо основного.

Плюсы

  • Маршруты не трогаем

Минусы

  • Два сайдбара на экране или хак с CSS
  • Грязное решение
